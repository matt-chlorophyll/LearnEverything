# 深度提问策略

本文件定义阶段 3 中**AI 主动发起的有限深度追问**。在验证问题回答后、开放讨论窗口之前执行。

---

## 核心原则

深度提问 ≠ 开放讨论。深度提问是 **AI 主导、有明确轮次上限** 的追问，目的是探测理解深度。开放讨论窗口在深度提问完成后才开始，由用户主导、无轮数限制。

### 阶段 3 的完整流程

```
AI 提验证问题
  → 用户回答
  → AI 深度追问（有限，本文件定义）
  → AI 解析总结
  → 开放讨论窗口（用户主导，无限）
  → 下一题
```

---

## 四种深度追问策略

### 1. 追问链

**定义**：基于用户回答继续深挖，逐层逼近理解边界。

**提问模式**：
- "你说X，那如果Y呢？"
- "为什么是这样而不是那样？"
- "这个推理的前提是什么？"

**深度限制**：最多 3 层追问。

**退出条件**：
- 用户触及理解边界（开始不确定或答错）→ 记录边界位置，停止追问
- 3 层全部通过 → 标记为深度理解，停止追问

### 2. 反例挑战

**定义**：给出一个似是而非的陈述，看用户能否识别错误。

**提问模式**：
- "有人说'{包含微妙错误的陈述}'，你觉得这个说法对吗？"
- "如果有人主张'{看似合理但有缺陷的观点}'，你会怎么回应？"

**适用场景**：概念类和机制类知识，尤其适合容易产生常见误解的概念。

**深度限制**：每轮验证最多 1 个反例挑战。

**退出条件**：
- 用户独立识别出错误 → 记录为"独立捕捉反例"
- 用户未识别 → 提示关键矛盾点，引导发现，不计为额外加分

### 3. 边界探测

**定义**：探索概念的适用边界和失效条件。

**提问模式**：
- "这在什么情况下不成立？"
- "这个概念的适用范围是什么？边界在哪？"
- "有没有例外情况？"

**适用场景**：已有明确适用范围或已知例外的概念。

**深度限制**：1-2 个边界问题。

**退出条件**：
- 用户能描述出合理的边界 → 正常结束
- 用户无法想到边界 → AI 补充一个关键边界案例，不强制继续

### 4. 元认知提问

**定义**：引导用户反思自己的理解过程和确定程度。

**提问模式**：
- "你觉得你对这个概念最不确定的部分是什么？"
- "如果要你教别人这个概念，你觉得最难讲清楚的是哪里？"

**使用频率**：每 session 最多 1 次，通常在第 2-3 个概念后使用。

**退出条件**：用户回答后，AI 针对其不确定点做简要补充，不继续追问。

## 策略选择逻辑

根据用户对主验证问题的回答质量，选择不同的深度追问组合：

```
用户回答主验证问题
  │
  ├── pass（准确且有深度）
  │     → 追问链（1-2 层，测试是否真正深入）
  │     → 反例挑战（适用于概念类/机制类）
  │     → 可选：边界探测（概念有明确边界时）
  │
  ├── partial（方向正确但不完整）
  │     → 追问链（1 层，定位具体缺口）
  │     → 跳过反例挑战和边界探测（避免打击信心）
  │
  └── fail（错误或无法回答）
        → 不使用深度追问策略
        → 直接进入引导重讲流程（现有机制）
```

### 元认知提问的插入时机

不依赖回答质量，独立判断：
- 每 session 的第 2-3 个概念时使用 1 次
- 在深度追问的最后（追问链/反例之后）插入
- 如果本 session 只学 1 个概念，在练习阶段后使用

---

## Confidence 加分

深度追问表现优秀时，给予额外 confidence 奖励：

| 表现 | 加分 |
|------|------|
| 追问链 3 层全部通过 | +0.1 |
| 独立捕捉反例中的错误 | +0.1 |
| 准确描述概念边界 | 不额外加分（已含在 pass 评价中） |

**每概念深度追问加分上限**：+0.1（即使追问链和反例都优秀，也只加一次）

---

## 执行注意事项

1. **自然过渡**：深度追问应从用户的回答自然延伸，不要让用户感觉在"考试"
2. **及时止步**：一旦触及理解边界就停止，不要把用户逼到无话可说
3. **鼓励氛围**：即使追问中暴露了不足，也要肯定用户思考的深度
4. **记录边界**：深度追问中发现的理解边界是宝贵的诊断信息，记录到讨论记录中
5. **画像参考**：如果 `learner-profile.json` 显示 `challengeAppetite: high`，可适当增加追问层数和反例难度；`low` 则减少
