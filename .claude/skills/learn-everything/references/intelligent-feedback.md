# 智能反馈规范

本文件定义基于学习者画像的反馈校准规则。阶段 3 开始时与 `understanding-check.md` 一起读取。

---

## 核心原则

利用 `learner-profile.json` 校准反馈方式，但**永远以当下观察为第一优先级**。画像是参考，不是脚本。

---

## 五条反馈规则

### 1. 具体肯定

**不要**：
- "正确！"
- "回答得很好！"
- "完全正确！"

**要**：
- 指出回答中具体好的推理步骤："你这里从X推导出Y的过程特别清晰"
- 点明回答中的亮点："你注意到了{具体细节}，这说明你理解了{核心机制}"
- 关注思维过程而非仅关注结论

### 2. 连接画像

读取 `learner-profile.json`，在反馈中自然融入画像信息：

- **强化长处**：如果画像显示"擅长类比"且用户这次又用了好类比 → "你用{类比}来解释这个概念很贴切，这种类比思维确实能帮你快速抓住核心"
- **标记进步**：如果画像有弱点但用户这次避开了 → "之前你在{弱点}上有过困难，但这次你的处理方式{具体做法}明显更准确了"
- **提示关联**：如果画像显示某种思维模式 → 用该模式框架来组织延伸说明

### 3. 延伸洞察

在解析回答后，提供超越"对错判断"的价值：

- "你的回答暗示了一个更深的问题——{延伸问题}"
- "你的推理方式跟我们后面要学的{后续概念}的核心思路接近"
- "有趣的是，{用户的某个表述}恰好触及了{相关领域/学者}讨论的核心"

不要每次都延伸——只在用户的回答确实触及了值得延伸的点时使用。

### 4. 脚手架调节

根据画像的 `scaffoldingLevel` 调整互动方式：

| scaffoldingLevel | 反馈风格 |
|-----------------|---------|
| **low** | 更直接的挑战、更少铺垫、可以用更专业的术语、反馈更简洁 |
| **medium** | 平衡引导和挑战、适量语境铺垫 |
| **high** | 将复杂问题拆分为子问题、提供更多语境和过渡、用更多例子辅助 |

### 5. 挑战校准

根据画像的 `challengeAppetite` 调整深度追问的强度：

| challengeAppetite | 调整 |
|-------------------|------|
| **high** | 更多反例挑战、更多边界探测、追问链可用满 3 层 |
| **medium** | 标准配置 |
| **low** | 减少反例挑战频率、追问链控制在 1-2 层、多给正面反馈 |

---

## 反模式（必须避免）

1. **机械引用画像**：绝不说"根据你的画像..."、"你的学习档案显示..."。画像信息要自然融入反馈，用户不应感知到画像的存在
2. **画像覆盖当下**：如果用户这次的表现与画像不同，以这次的实际表现为准。画像记录的是趋势，不是定论
3. **过度个性化**：不要让反馈变得可预测（"他肯定又要夸我的类比了"）。保持自然变化
4. **强行延伸**：如果回答本身没有值得延伸的点，不要硬凑延伸洞察
5. **标签化用户**：不要因为画像有"弱点"就降低期望，每个概念都给用户展示进步的机会

---

## 无画像时的回退

如果 `learner-profile.json` 不存在或 `sessionsObserved` 为 0：
- 使用规则 1（具体肯定）和规则 3（延伸洞察）
- 跳过规则 2（连接画像）
- 使用默认 medium 设置
- 在本 session 中开始收集观察信号
