# 学习文档生成规范

本文件定义阶段 2（文档生成）的执行逻辑。目标：生成聚焦单一概念的高质量学习文档。

---

## 概念选择策略

基于知识图谱选择下一个学习概念，优先级从高到低：

1. **weak 且被依赖** — 基础薄弱影响后续学习，优先修复
2. **weak 非依赖** — 已暴露的弱点需要巩固
3. **unknown 且依赖已满足** — 前置知识已具备，可以自然推进
4. **unknown 且无依赖** — 独立概念，随时可学
5. **undiscovered** — 边界扫描时发现的新概念

如果有多个同优先级概念，选择与最近学习内容关联度最高的（基于 `relates-to` 和 `extends` 边）。

---

## 文档结构模板

每篇文档遵循以下结构，但根据概念特点灵活调整比重：

### 1. 定义（什么是 X）
- 一句话精确定义
- 如有必要，补充一个直觉类比

### 2. 动机（为什么需要 X）
- 没有 X 会怎样？X 解决了什么问题？
- 与用户已掌握的概念建立连接："你已经知道了 Y，X 是在此基础上..."

### 3. 原理（X 如何工作）
- 核心机制的逐步解释
- 适当使用示例辅助说明
- 概念类知识侧重逻辑链条；技能类知识侧重操作步骤

### 4. 关联（X 与其他概念的关系）
- 与已学概念的关系（requires / extends / relates-to）
- 简要提及后续可能学到的概念（预告但不展开）

### 5. 边界（X 的适用范围和常见误解）
- X 不是什么（常见混淆点）
- X 在什么情况下不适用
- 仅在概念有明确边界时包含此节，不要生造

## 质量标准

### 篇幅
- 目标 500-1000 字
- 基础概念可短些（500 字左右）
- 复杂概念可长些（接近 1000 字）
- 绝不超过 1200 字

### 语言风格
- 对话式口吻，像在跟用户面对面讲解
- 避免学术论文式的冗长表述
- 用"你"而非"读者"

### 根据用户水平调整

| 用户水平 | 调整策略 |
|----------|---------|
| 完全新手 | 多用类比和日常例子，避免术语，定义部分加厚 |
| 有基础 | 可使用领域术语，侧重原理和关联 |
| 有经验 | 跳过基础定义，聚焦边界、深层机制和常见误解 |

水平判断依据：阶段 1 诊断结果 + 知识图谱中已掌握概念的数量和层次。

### 参考学习者画像

如果 `domains/{domain}/learner-profile.json` 存在且 `sessionsObserved > 0`：
- **思维模式适配**：如果用户偏好类比思维，文档中增加高质量类比；偏好结构化分析则用更清晰的层次结构
- **脚手架调节**：`scaffoldingLevel` 为 high 时，拆分更细、例子更多；为 low 时，文档更紧凑、直接进入核心
- **已知弱点关注**：如果画像记录了与当前概念相关的弱点，在文档中针对性加强该部分的解释

## 文件保存规范

### 文件名
`{序号}-{概念名}.md`

- 序号：两位数字，从 `01` 开始，按生成顺序递增
- 概念名：英文小写，用 `-` 连接
- 示例：`01-variable.md`、`02-data-type.md`、`15-binary-search.md`

### 序号确定
读取 `domains/{domain}/docs/` 目录，找到当前最大序号，加 1。如果目录为空，从 `01` 开始。

### 保存路径
`domains/{domain-name}/docs/{序号}-{概念名}.md`

### 文档头部元信息
在文档最开头添加 YAML front matter：

```markdown
---
concept: "{概念英文 id}"
domain: "{领域名}"
date: "{YYYY-MM-DD}"
prerequisites: ["{前置概念 id}"]
---
```

---

## 执行步骤

1. 根据概念选择策略确定本次教学概念
2. 将该概念在知识图谱中标记为 `learning`，更新 `meta.json` 的 `currentFocus`
3. 根据文档结构模板生成文档内容
4. 保存文档到指定路径
5. 将文档内容直接展示给用户（不要只告诉用户文件路径）
6. 等待用户阅读确认后进入阶段 3
