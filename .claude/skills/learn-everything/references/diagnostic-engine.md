# 诊断提问策略

本文件定义阶段 1（诊断）的执行逻辑。目标：快速准确地探测用户当前知识水平。

---

## 冷启动诊断（新领域 / 首次 session）

### 策略：广→窄、浅→深

设计 5-8 个渐进式问题，按以下层次递进：

**第 1-2 题：广域探测**
- 开放式问题，了解用户对该领域的整体认知
- 示例格式："你对{领域}了解多少？能简单说说你知道的吗？"
- 目的：判断是完全新手、有些基础、还是有经验

**第 3-4 题：核心概念定位**
- 根据前两题的回答，针对该领域的核心概念提问
- 示例格式："你能解释一下{核心概念}是什么吗？" / "你觉得{A}和{B}有什么区别？"
- 目的：确定用户在核心概念上的掌握程度

**第 5-6 题：边界测试**
- 测试用户知识的边界——他们在哪里开始不确定
- 示例格式：给出一个需要综合运用的场景，问用户如何处理
- 目的：找到 known-unknown 的边界

**第 7-8 题（可选）：深度探测**
- 只在用户表现出较高水平时使用
- 针对进阶概念或容易混淆的点提问
- 目的：精确定位起始教学点

### 动态调整规则

- 如果前 2 题用户回答为"完全不了解"，跳过后续深度问题，直接从最基础概念开始
- 如果用户在某题表现出明确的知识断层，记录该断层点，不再继续深入该方向
- 每题根据上一题回答调整方向，不要机械执行预设问题列表

## 继续学习诊断（已有 session 历史）

### 策略：记忆确认 + 衰减检测

设计 2-3 个问题：

**第 1 题：最近概念回顾**
- 针对上次 session 学习的核心概念提问
- "上次我们学了{概念}，你能回忆一下它的核心要点吗？"

**第 2 题：应用检验**
- 给出一个简单场景，看用户能否应用上次学的知识
- 目的：检测是否从"理解"退化到"模糊记忆"

**第 3 题（可选）：关联检查**
- 如果用户上次学了多个概念，检查概念间的关联理解
- 目的：确认知识网络是否稳固

### 记忆衰减处理

- 如果用户遗忘严重（核心要点无法回忆）：将该概念状态从 mastered 降为 weak，在阶段 2 优先复习
- 如果用户记忆模糊但方向正确：保持状态，在文档中简要回顾后继续新概念
- 如果用户记忆清晰：确认后直接进入新概念学习

## 问题类型工具箱

根据领域特点混合使用：

| 类型 | 适用场景 | 格式 |
|------|---------|------|
| **开放式探测** | 了解整体认知 | "你对X了解多少？" / "说说你理解的X" |
| **边界测试** | 定位知识断层 | "X和Y的区别是？" / "X在什么情况下不适用？" |
| **关联检查** | 测试知识网络 | "X和Y有什么关系？" / "学了X之后，你觉得Z会怎样？" |
| **场景应用** | 测试实际运用 | "如果遇到{场景}，你会怎么处理？" |
| **概念辨析** | 检测精确理解 | "有人说{常见误解}，你怎么看？" |

## 诊断结果映射

诊断完成后，将结果映射到知识图谱：

1. **用户展示掌握的概念** → 添加节点，状态 `mastered`，confidence 按回答质量设定（0.6-0.9）
2. **用户听说过但解释不清的概念** → 添加节点，状态 `weak`，confidence 0.2-0.4
3. **用户完全不了解的概念** → 添加节点，状态 `unknown`，confidence 0.0
4. **诊断中发现的关联概念**（用户未提及但存在依赖关系）→ 添加节点，状态 `undiscovered`

### 确定起始教学概念

从知识图谱中选择第一个教学概念，优先级：
1. `weak` 状态 + 被多个概念依赖的节点（修复基础）
2. `unknown` 状态 + 依赖已满足的节点（自然推进）
3. 全新基础概念（如果用户是完全新手）
